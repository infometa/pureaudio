# 第一句话被吞掉问题审计报告

## 问题描述

用户反馈：点击开始降噪后，第一句话大概率会被吞掉。

## 问题分析

### 根本原因分析

经过代码审计，发现以下**多个叠加因素**导致第一句话被吞掉：

---

## 1. 🔴 严重问题：启动淡入（Fade-in）导致开头被衰减

**位置**: `capture.rs:1464-1476`

**问题代码**:
```rust
// 启动淡入，避免播放砰声
if fade_progress < fade_total {
    if let Some(buffer) = outframe.as_slice_mut() {
        for v in buffer.iter_mut() {
            if fade_progress >= fade_total {
                break;
            }
            let g = (fade_progress as f32 / fade_total as f32).min(1.0);
            *v *= g;  // 前80ms的输出被线性衰减
            fade_progress += 1;
        }
    }
}
```

**问题分析**:
1. **淡入长度**: 80ms (`fade_total = sr * 0.08`)
2. **淡入时机**: 在**最终限幅之后**应用，这意味着：
   - 所有处理（降噪、EQ、AGC等）都已经完成
   - 但输出被线性衰减，前80ms几乎听不到
3. **线性淡入**: 使用线性淡入，前几帧的增益非常小（接近0）

**影响**:
- **前80ms的输出几乎被完全衰减**
- 如果用户在第一句话的前80ms内开始说话，开头会被吞掉
- 第一句话的开头（通常是起音）被衰减，导致听不清

**优先级**: 🔴 **最高** - 这是最直接的原因

---

## 2. 🔴 严重问题：VAD初始化导致第一句话被误判为噪声

**位置**: `capture.rs:1114-1126, 1309-1318`

**问题代码**:
```rust
// VAD 初始状态
let mut vad_state = false;
let mut vad_voice_count = 0usize;
let mut vad_noise_count = 0usize;

// VAD 判定逻辑
if is_voice {
    vad_voice_count = vad_voice_count.saturating_add(1).min(50);
    vad_noise_count = vad_noise_count.saturating_sub(vad_noise_count.min(1));
} else {
    vad_noise_count = vad_noise_count.saturating_add(1).min(50);
    vad_voice_count = vad_voice_count.saturating_sub(vad_voice_count.min(1));
}
if vad_voice_count >= 3 {
    vad_state = true;
} else if vad_noise_count >= 2 {
    vad_state = false;
}
is_voice = vad_state;

// 非语音段：重度抑制
if !is_voice {
    target_atten = (target_atten + 22.0).min(100.0);  // 增加22dB抑制！
    target_hp = target_hp.max(220.0);  // 高通提高到220Hz
    target_min_thresh = target_min_thresh.max(-54.0);
    target_max_thresh = target_max_thresh.min(6.0);
    target_exciter_mix = 0.0;
}
```

**问题分析**:
1. **VAD初始化**: `vad_state = false`，初始状态是噪声
2. **判定延迟**: 需要**3帧**才能判定为语音（`vad_voice_count >= 3`）
3. **误判后果**: 在第一句话的前几帧，VAD可能判定为噪声，触发**重度抑制**：
   - 衰减增加22dB
   - 高通提高到220Hz（可能切掉语音低频）
   - 阈值提高，降噪更激进

**影响**:
- **第一句话的前几帧被重度抑制**
- 如果第一句话的开头能量较低，更容易被误判为噪声
- 叠加淡入效果，第一句话几乎听不到

**优先级**: 🔴 **最高** - 这是第二个主要原因

---

## 3. 🟡 中等问题：环境自适应初始参数导致过度抑制

**位置**: `capture.rs:856-876`

**问题代码**:
```rust
// 初始参数
let mut env_class = EnvClass::Noisy;  // 初始分类为"嘈杂"
let mut smoothed_energy = -80.0f32;  // 初始能量很低
let mut noise_floor_db = -60.0f32;   // 初始噪声地板
let mut snr_db = 10.0f32;             // 初始SNR
let mut target_atten = 45.0f32;       // 初始衰减45dB
let mut target_hp = 80.0f32;          // 初始高通80Hz
```

**问题分析**:
1. **初始分类**: `EnvClass::Noisy`（嘈杂），导致参数偏向高抑制
2. **SNR计算**: 初始SNR可能不准确，因为噪声地板是估计值
3. **参数映射**: 如果SNR < 10.0，会强制偏向"办公区"模式，衰减更高

**影响**:
- 启动时可能使用过度抑制的参数
- 第一句话可能被过度降噪

**优先级**: 🟡 中

---

## 4. 🟡 中等问题：噪声地板初始化不准确

**位置**: `capture.rs:869, 1133-1148`

**问题代码**:
```rust
let mut noise_floor_db = -60.0f32;  // 初始噪声地板

// 噪声地板跟踪：仅在非语音段更新
if !is_voice {
    if rms_db < noise_floor_db {
        noise_floor_db = smooth_value(noise_floor_db, rms_db, 0.45);
    } else {
        noise_floor_db = smooth_value(noise_floor_db, rms_db, 0.03);
    }
}
snr_db = (rms_db - noise_floor_db).clamp(-5.0, 30.0);
```

**问题分析**:
1. **初始值**: -60dB 是固定值，可能不准确
2. **更新条件**: 只在非语音段更新，但启动时可能误判为语音
3. **SNR计算**: 如果噪声地板不准确，SNR计算错误，导致参数调整错误

**影响**:
- SNR计算可能不准确
- 环境自适应参数可能错误

**优先级**: 🟡 中

---

## 5. 🟢 轻微问题：启动保护期未使用

**位置**: `capture.rs:891-892`

**问题代码**:
```rust
// 启动保护期，前 1s 禁止 gate/瞬态强抑制，避免开口被吞
let startup_guard_until = Instant::now() + Duration::from_millis(1000);
```

**问题分析**:
1. **定义了保护期**: `startup_guard_until` 被定义，但**代码中没有使用**
2. **保护期失效**: 虽然注释说"避免开口被吞"，但实际没有生效

**影响**:
- 启动保护期没有生效
- 第一句话可能被误判和处理

**优先级**: 🟢 低（但修复简单）

---

## 6. 🟢 轻微问题：AGC初始化可能需要时间稳定

**位置**: `agc.rs:14-50`

**问题分析**:
1. **WebRTC AGC**: 使用WebRTC的AGC，可能需要时间稳定
2. **初始化**: AGC初始化时可能增益较低，需要时间调整

**影响**:
- 第一句话可能增益不足
- 但影响相对较小

**优先级**: 🟢 低

## 问题叠加效应

### 叠加时间线

```
时间轴（第一句话开始）:
0ms    20ms   40ms   60ms   80ms   100ms  120ms
|------|------|------|------|------|------|
淡入衰减 100%  50%   25%   12%   6%    0%    ← 淡入效果
VAD误判  噪声  噪声  噪声  语音  语音  语音  ← VAD判定延迟
抑制    重度  重度  重度  正常  正常  正常  ← 环境自适应
```

**叠加效果**:
- **0-60ms**: 淡入衰减 + VAD误判 + 重度抑制 = **几乎完全吞掉**
- **60-80ms**: 淡入衰减 + VAD误判 + 重度抑制 = **严重衰减**
- **80-120ms**: 淡入结束 + VAD判定为语音 + 正常抑制 = **开始恢复**

**结论**: 第一句话的前**80-120ms**被严重衰减或吞掉！

## 解决方案

### 🔴 方案1：修复启动淡入（最高优先级）

**问题**: 淡入在最终限幅之后，导致开头被衰减

**解决方案**:
1. **移除淡入**（最简单）
2. **缩短淡入时间**（如20ms）
3. **使用指数淡入**替代线性淡入
4. **淡入移到处理前**（但可能影响处理效果）

**推荐**: 缩短淡入时间到20ms，或使用指数淡入

**代码修改**:
```rust
// 方案A: 缩短淡入时间
let fade_total = (df.sr as f32 * 0.02) as usize; // 20ms 而不是 80ms

// 方案B: 使用指数淡入
let g = 1.0 - (-fade_progress as f32 / (fade_total as f32 / 3.0)).exp();
```

---

### 🔴 方案2：修复VAD初始化（最高优先级）

**问题**: VAD初始状态为噪声，需要3帧才能判定为语音

**解决方案**:
1. **启动保护期内强制判定为语音**
2. **降低VAD判定阈值**（启动时）
3. **启动时禁用VAD**（前1秒）
4. **使用启动保护期**

**推荐**: 在启动保护期内强制判定为语音，或降低判定阈值

**代码修改**:
```rust
// 在启动保护期内
if Instant::now() < startup_guard_until {
    // 强制判定为语音，避免误判
    is_voice = true;
    vad_state = true;
    vad_voice_count = 3;  // 直接设置为3，跳过判定延迟
} else {
    // 正常VAD判定逻辑
    // ...
}
```

---

### 🟡 方案3：优化环境自适应初始参数

**问题**: 初始参数可能导致过度抑制

**解决方案**:
1. **启动时使用保守参数**（低抑制）
2. **延迟环境自适应**（启动后1秒再启用）
3. **使用启动保护期**

**推荐**: 启动时使用保守参数，1秒后再启用环境自适应

**代码修改**:
```rust
if Instant::now() < startup_guard_until {
    // 启动保护期：使用保守参数
    target_atten = 30.0;  // 降低抑制
    target_hp = 60.0;     // 降低高通
    target_min_thresh = -60.0;  // 降低阈值
    target_max_thresh = 12.0;
} else {
    // 正常环境自适应逻辑
    // ...
}
```

---

### 🟡 方案4：优化噪声地板初始化

**问题**: 初始噪声地板可能不准确

**解决方案**:
1. **启动时快速学习噪声地板**（前几帧）
2. **使用更保守的初始值**
3. **延迟SNR计算**（启动后）

**推荐**: 启动时快速学习噪声地板

---

### 🟢 方案5：启用启动保护期

**问题**: 定义了但未使用

**解决方案**: 在关键位置检查 `startup_guard_until`

**代码修改**:
```rust
// 在VAD判定处
if Instant::now() < startup_guard_until {
    is_voice = true;  // 强制判定为语音
}

// 在环境自适应处
if Instant::now() < startup_guard_until {
    // 使用保守参数
}
```

## 总结

**根本原因**:
1. **启动淡入80ms**导致开头被衰减（主要原因）
2. **VAD初始化延迟**导致第一句话被误判为噪声（主要原因）
3. **环境自适应初始参数**可能导致过度抑制（次要原因）

**修复优先级**:
1. 🔴 **立即修复启动淡入**（缩短到20ms或使用指数淡入）
2. 🔴 **立即修复VAD初始化**（启动保护期内强制判定为语音）
3. 🟡 **启用启动保护期**（在关键位置检查）

**预期效果**: 修复后，第一句话应该**完整保留**，不会被吞掉。
